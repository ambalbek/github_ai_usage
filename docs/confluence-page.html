1. Overview

The GitHub Copilot AI Credits / Premium Request Billing Exporter is a Python-based Prometheus exporter that collects billing and usage data from the GitHub Enhanced Billing Platform API and exposes it as Prometheus metrics on port 9185. It optionally indexes the same data into Elasticsearch for long-term storage and Kibana dashboards.

It answers one question: How much are our GitHub Copilot AI features costing us — by model, by SKU, by month, across all our orgs and enterprises?

Key Capabilities

  • Multi-entity support — monitor multiple GitHub organizations and enterprises from a single exporter instance
  • Three-endpoint coverage — queries ai_credit/usage, premium_request/usage, and usage APIs to capture both new and legacy billing models
  • Intelligent deduplication — merges overlapping data from multiple endpoints, keeping the highest values
  • TTL-based caching — avoids hammering the GitHub API on every Prometheus scrape (default 15 min cache)
  • Multi-month history — fetches current + N previous months of billing data for trend analysis
  • Dual output — Prometheus metrics for real-time alerting + Elasticsearch for historical analytics
  • Pre-built dashboards — Grafana (7 rows, 28 panels) and Kibana dashboards included
  • Production-ready — Helm chart with Flux-compatible deployment, ServiceMonitor, health probes, non-root container

3.2 Per-Entity Fetch Strategy (Three-Endpoint Merge)

For each (entity, month) pair, the exporter calls all three GitHub billing endpoints and merges the results. This ensures complete coverage across billing model transitions:

text Three-Endpoint Fetch & Merge Strategy

Why three endpoints? GitHub is transitioning billing models. The ai_credit/usage endpoint was introduced in June 2026 for the new AI Credits model. The premium_request/usage endpoint serves legacy annual plans. The general usage endpoint is a fallback that covers both but with less granularity (no gross/net split — only quantity). The exporter calls all three and deduplicates to ensure no data is missed during transition periods.

3.3 Multi-Month Fetching

The months_back config parameter (default: 11) controls how many months of history are fetched. With months_back=11, the exporter fetches the current month plus the 11 previous months (a full rolling year).

17. Troubleshooting

SymptomLikely CauseResolution
scrape_success=0 All API calls failed (auth, network, or wrong entity slugs) Check logs for 401/403/404. Verify token permissions and org/enterprise slugs.
No metrics for some models Entity not yet on Enhanced Billing Platform, or billing model transition Ensure the org/enterprise is enrolled in GitHub's Enhanced Billing. Check all three endpoint logs.
Seat license costs appearing exclude_skus not configured or custom SKU names Add the SKU names to exclude_skus in config.json (case-insensitive).
High scrape duration Many entities × many months = many API calls Reduce months_back, increase cache_ttl_seconds, or reduce entity count.
ES indexing failures Wrong URL, expired API key, index permission Check ELASTICSEARCH_URL and ES_API_KEY. Verify index write permissions. Failures are logged but don't affect Prometheus metrics.
Rate limiting (429) Too many API calls to GitHub Increase cache_ttl_seconds to reduce call frequency. The exporter logs the rate limit reset timestamp.
